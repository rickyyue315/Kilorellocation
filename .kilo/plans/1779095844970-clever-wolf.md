# Plan: F2 Mode - Windy Source Priority for Windy Destinations

## Problem

In F2 mode, when a Windy shop has a Target (is a destination), the current sorting logic does NOT guarantee that Windy sources are always prioritized over non-Windy sources. Specifically:

- The current `_sort_key` in `strategies/f_mode.py:44-53` uses `(tier + hd_penalty, effective_sold_qty)`
- For a Windy destination: non-Windy ND sources (tier=0) are sorted BEFORE Windy RF sources (tier=1)
- This means non-Windy ND sources can be consumed before Windy sources, which is undesirable

## Desired Behavior

When a Windy shop is a destination (has Target > 0) in F2 mode:
1. **First**: Extract from other Windy shops without Target (Windy sources)
2. **Then**: If Windy sources are insufficient, fall back to non-Windy sources

## Implementation

### File: `strategies/f_mode.py`

Modify the `_sort_key` function (line 44-53) to add a Windy priority bonus:

- Add a `windy_penalty` variable
- When `dest_om == 'Windy'` and `src.get('om') != 'Windy'`, set `windy_penalty = 5` (enough to push non-Windy sources after all Windy sources in sort order)
- Include `windy_penalty` in the sort key return value

**Modified `_sort_key`:**
```python
def _sort_key(src, dest_om, dest_site=''):
    rp = str(src.get('rp_type', '')).upper()
    same_om = 1 if src.get('om') == dest_om else 2
    tier = 0 if rp == 'ND' else (1 if same_om == 1 else 2)
    hd_penalty = 0
    if (is_f2 and self._f2_allow_hd_transfer
            and str(src.get('site', '')).upper().startswith('HD')
            and dest_site.upper().startswith(('HA', 'HB', 'HC'))):
        hd_penalty = 10
    windy_penalty = 0
    if is_f2 and dest_om == 'Windy' and src.get('om') != 'Windy':
        windy_penalty = 5
    return (tier + hd_penalty + windy_penalty, src.get('effective_sold_qty', 0))
```
