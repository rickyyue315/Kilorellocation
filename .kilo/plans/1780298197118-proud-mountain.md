# AI_INTEGRATION_PLAN 詳細實施計劃

## 0. 目標與範圍

依據 `plans/AI_INTEGRATION_PLAN.md`，為現有 Streamlit 庫存調貨建議系統加入三個非阻塞 AI 場景：

1. **模式推薦 / AI Advisor**：上傳並預處理資料後，根據統計摘要推薦合適調貨模式；不自動切換使用者選擇。
2. **邏輯審計 / AI Auditor**：調貨建議及既有 quality checks 完成後，提供風險提示；不修改建議結果。
3. **報表增強 / AI Report Enhancement**：優先新增 Excel `AI分析摘要` sheet；不優先覆寫逐行 `Notes`，避免降低既有確定性註解品質。

核心原則：

- AI 只能提供建議與摘要，**不得改變 `business_logic.py` 的 deterministic 調貨結果**。
- 預設行為必須安全：缺少 API key、AI 關閉、API 失敗、JSON 解析失敗時，核心調貨流程仍完整可用。
- 不上傳原始 DataFrame 全量資料；只傳 aggregate summary、統計、quality errors、必要且 capped 的推薦摘要。
- 不記錄 API key、不在 UI 或 log 暴露 prompt 全量敏感內容。
- AI integration 不是「模式邏輯修改」，因此不觸發 AGENTS.md 的 transfer mode documentation sync 清單；但若 bump 版本，需同步 `config.py`、`app.py` docstring、`VERSION.md`、`README.md`。

## 1. 現況觀察與主要接點

### 1.1 已確認檔案

- `plans/AI_INTEGRATION_PLAN.md`：原始 AI 整合構想。
- `app.py`：Streamlit 主流程，主要插入點：
  - `render_sidebar()` 在 line 57 左右已於上傳前執行。
  - `_cached_preprocess(uploaded_file.getvalue())` 在 line 111 左右完成資料預處理。
  - `render_data_preview(...)` 在 line 157 左右。
  - `generate_transfer_recommendations(...)`、`perform_quality_checks(...)`、`get_transfer_statistics()` 在 line 172-193 左右。
  - Excel 生成在 line 223-228 左右。
- `ui/sidebar.py`：模式 radio 在 line 103-145 左右；因 sidebar 在 upload 前渲染，不能在同一次 render 直接取得剛上傳 DataFrame。
- `ui/display.py`：適合新增 AI UI helper，例如 `render_ai_advisor_card(...)`、`render_ai_audit_report(...)`。
- `excel_generator.py`：目前只產生 `調貨建議 (Transfer Recommendations)`、`統計摘要 (Summary Dashboard)`；適合新增 optional `AI分析摘要` sheet。
- `config.py`：已有 `os` import 與 runtime config helper；適合放 AI defaults 與 env parsing。
- `models/mode_registry.py`：`MODE_DEFS` 是 advisor 可用的模式來源，不應手寫重複模式清單。
- `services/notes.py`：現有 Notes 已有 deterministic、結構化註解。
- `tests/test_ai_note_simulation.py`：已指出逐行 AI Notes 多半只增加文字、不增加資訊，應降低優先級。
- `requirements.txt`：目前沒有 HTTP client；需新增 `httpx>=0.27.0`。

### 1.2 架構決策

原 plan 提到在 `ui/sidebar.py` radio 上方顯示 AI 模式推薦，但現有 `app.py` 先 render sidebar、後 upload/preprocess。建議採用兩階段安全方案：

- **MVP 顯示位置**：先在主頁 `render_data_preview(...)` 之後、`分析與建議` 之前顯示 AI Advisor card，因為此時已有 DataFrame summary。
- **Sidebar 補充顯示**：如需在 sidebar 呈現，只讀取 `st.session_state['ai_advisor_result']`，在下一次 rerun 顯示「上一份已分析資料的 AI 建議」。不要為了 sidebar 位置重構整個主流程。

## 2. 實施總順序

建議以小步提交/驗證方式進行：

1. AI 基礎配置與 client，不接入 UI。
2. AI Advisor summary/prompt/parser + 主頁 UI。
3. AI Auditor summary/prompt/parser + 結果區 UI。
4. Excel `AI分析摘要` sheet，使用 auditor/advisor 結果及統計摘要。
5. 可選：row-level AI notes enhancer，預設關閉；只有確認有業務價值才啟用。
6. 文件、版本、測試、部署說明收尾。

## 3. Phase 1 — AI 基礎建設

### 3.1 修改 `config.py`

新增 AI config，放在 Zeabur config 之後或 Magic Numbers 前。使用小型 helper 解析 bool/int，避免 env 值錯誤導致 crash。

建議常數：

```python
AI_DEFAULT_MODEL = os.getenv('AI_MODEL', 'deepseek/deepseek-v4-flash')
AI_MODEL_ADVISOR = os.getenv('AI_MODEL_ADVISOR', AI_DEFAULT_MODEL)
AI_MODEL_AUDITOR = os.getenv('AI_MODEL_AUDITOR', AI_DEFAULT_MODEL)
AI_MODEL_ENHANCER = os.getenv('AI_MODEL_ENHANCER', AI_DEFAULT_MODEL)
AI_ENABLED = _get_env_bool('AI_ENABLED', False)
AI_REQUEST_TIMEOUT = _get_env_int('AI_REQUEST_TIMEOUT', 30)
AI_MAX_TOKENS_ADVISOR = _get_env_int('AI_MAX_TOKENS_ADVISOR', 512)
AI_MAX_TOKENS_AUDITOR = _get_env_int('AI_MAX_TOKENS_AUDITOR', 1024)
AI_MAX_TOKENS_REPORT = _get_env_int('AI_MAX_TOKENS_REPORT', 1024)
AI_ENHANCE_NOTES_ENABLED = _get_env_bool('AI_ENHANCE_NOTES_ENABLED', False)
AI_ENHANCE_MIN_QTY = _get_env_int('AI_ENHANCE_MIN_QTY', 10)
AI_ENHANCE_MAX_ROWS = _get_env_int('AI_ENHANCE_MAX_ROWS', 30)
```

重要：建議 `AI_ENABLED` 預設 `False`，避免本地測試或 Zeabur 未設定 secret 時誤觸外部 API。若希望部署時預設啟用，可在 Zeabur 設 `AI_ENABLED=true`。

### 3.2 修改 `requirements.txt`

新增：

```text
httpx>=0.27.0
```

### 3.3 新增 `services/ai_client.py`

職責：封裝 OpenRouter REST API、secret/env 讀取、session cache、錯誤降級。

必要函式：

- `get_secret_or_env(name: str, default: str = '') -> str`
  - 優先 `st.secrets[name]`，失敗或非 Streamlit context 時 fallback 到 `os.getenv`。
- `is_ai_enabled() -> bool`
  - 同時考慮 `config.AI_ENABLED` 與 `AI_ENABLED` secret override。
- `get_ai_status() -> dict`
  - 回傳 `{enabled, has_api_key, model, reason}`，供 UI 顯示。
- `chat_completion(messages, model=None, temperature=0.1, max_tokens=1024, cache_namespace='default') -> str`
  - AI disabled / no key / exception 時回傳 `''`。
  - 使用 `httpx.Client(timeout=AI_REQUEST_TIMEOUT)`。
  - cache key 使用 `sha256(json.dumps(..., ensure_ascii=False, sort_keys=True))`。
  - cache 放在 `st.session_state.setdefault('_ai_cache', {})`；若不在 Streamlit context，使用模組內 fallback dict 或直接不 cache。

OpenRouter request headers：

```python
{
    'Authorization': f'Bearer {api_key}',
    'Content-Type': 'application/json',
    'HTTP-Referer': os.getenv('OPENROUTER_SITE_URL', ''),
    'X-Title': os.getenv('OPENROUTER_APP_TITLE', 'KiLo Reallocation'),
}
```

注意事項：

- log 只可記錄錯誤類型/status，不可記錄 API key、prompt、response full content。
- HTTP 4xx/5xx 不 raise 到 app。
- response parsing 要防守：missing `choices` 時回傳 `''`。

### 3.4 新增 tests

新增 `tests/test_ai_client.py`，不呼叫真 API。

測試項：

1. `AI_ENABLED=false` 時 `chat_completion(...) == ''`。
2. 缺少 API key 時回傳 `''`。
3. mock `httpx.Client.post` 成功時回傳 content。
4. mock HTTP exception 時回傳 `''`。
5. cache 命中時第二次不重複 post。
6. `_get_api_key` / `get_secret_or_env` 不因 `st.secrets` exception crash。

## 4. Phase 2 — AI Advisor（模式推薦）

### 4.1 新增 `services/ai_advisor.py`

建議資料結構：

```python
@dataclass(frozen=True)
class AIAdvisorResult:
    mode_code: str
    mode_name: str
    confidence: str
    reasons: list[str]
    warnings: list[str]
    raw_text: str = ''
```

若不想引入 dataclass 對 UI 轉換成本，也可回傳 plain dict，但測試需固定 schema。

### 4.2 Summary builder

新增 pure function：

```python
def build_df_summary(df: pd.DataFrame, processing_stats: dict | None = None) -> dict:
```

只輸出 aggregate：

- `total_rows`
- `unique_articles`
- `unique_oms`
- `unique_sites`
- `rp_type_counts`
- `nd_ratio`
- `rf_ratio`
- `zero_stock_count`
- `low_stock_count`（例如 `SaSa Net Stock + Pending Received <= 1`）
- `stock_over_safety_count`（可判斷 A/B 類需求）
- `has_type_column`、`type_counts`（cap top values）
- `has_target_column`、`target_positive_rows`、`target_positive_sites`
- `has_all_column`、`all_marked_rows`
- `cross_om_article_count`
- `om_distribution_top10`
- `site_count_by_om_top10`
- `invalid_rp_type_count` from `processing_stats`

不要傳：

- `Article Description` 明細。
- 原始檔名。
- 全量 row records。

### 4.3 Mode options builder

使用 `models.mode_registry.MODE_DEFS` 生成 allowed modes：

```python
def build_mode_options() -> list[dict]:
    return [
        {
            'code': d.code,
            'name': d.name,
            'description': d.description,
            'cross_om_matching': d.cross_om_matching,
            'receive_site_limit': d.receive_site_limit,
            'required_columns': sorted(d.required_columns),
        }
        for d in MODE_DEFS
    ]
```

### 4.4 Prompt 與 JSON parser

新增：

- `build_advisor_messages(summary: dict, mode_options: list[dict]) -> list[dict]`
- `parse_advisor_response(text: str) -> dict`
- `recommend_mode(df, processing_stats=None) -> dict`

要求模型只輸出 JSON：

```json
{
  "mode_code": "F2",
  "mode_name": "F指定模式",
  "confidence": "medium",
  "reasons": ["..."],
  "warnings": ["..."]
}
```

Parser 必須支援：

- fenced code block 中的 JSON。
- JSON 前後有文字時截取第一個 `{...}`。
- parse 失敗回傳 `{}` 或 `{'error': 'parse_failed', 'raw_text': text[:500]}`。

Validation：

- `mode_code` 必須存在於 `get_mode_def_by_code(mode_code)`。
- `mode_name` 以 registry 的 name 為準，不信任 LLM。
- `confidence` 只允許 `low|medium|high`，其他值降為 `low`。
- `reasons/warnings` cap 到最多 5 條，每條最多 160 字。

### 4.5 UI 整合

在 `ui/display.py` 新增：

```python
def render_ai_status_badge(status: dict): ...
def render_ai_advisor_card(result: dict, current_mode_code: str): ...
```

在 `app.py`：

1. import `recommend_mode`、`get_ai_status`、`render_ai_advisor_card`。
2. 在 `render_data_preview(df, processing_stats)` 後、`分析與建議` 前：
   - 建立 `data_signature`，例如 `uploaded_file.name + uploaded_file.size + processed rows + columns hash`。
   - 若 `st.session_state.get('ai_advisor_key') != data_signature`，清除舊 advisor result。
   - 顯示「AI 模式建議」區塊。
   - 建議不要每次自動呼叫；採用 button：`🤖 產生 AI 模式建議`。若產品要求自動，可只在首次 upload 後執行一次並 cache。
3. AI advisor result 寫入：
   - `st.session_state['ai_advisor_result']`
   - `st.session_state['ai_advisor_key']`

UI 文案：

- `AI 建議模式：F2: F指定模式`
- `信心：medium`
- `原因：...`
- `注意：AI 僅供參考，請仍以業務判斷選擇模式。`
- 若建議與目前 sidebar mode 不同，顯示：`目前選擇為 X，如需採用 AI 建議，請在左側手動切換。`

可選 sidebar 補充：

- 在 `ui/sidebar.py` radio 上方讀取 `st.session_state.get('ai_advisor_result')` 顯示簡短 caption。
- 不在 sidebar 內觸發 AI call。

## 5. Phase 3 — AI Auditor（邏輯審計）

### 5.1 新增 `services/ai_auditor.py`

主要函式：

- `build_audit_payload(recommendations, statistics, quality_passed, quality_errors, mode) -> dict`
- `build_auditor_messages(payload) -> list[dict]`
- `parse_audit_response(text) -> dict`
- `audit_recommendations(...) -> dict`

### 5.2 Audit payload 設計

只傳 summary + capped samples：

- `mode`
- `quality_passed`
- `quality_error_count`
- `quality_errors`：最多 10 條，每條 240 字。
- `total_recommendations`
- `total_transfer_qty`
- `unique_articles`
- `unique_oms`
- `source_type_stats`
- `dest_type_stats`
- `top_articles_by_qty`：最多 10。
- `top_transfer_sites_by_qty`：最多 10。
- `top_receive_sites_by_qty`：最多 10。
- `cross_om_transfer_count`。
- `large_qty_recommendations`：最多 10 條，只包含 `Article`, `Transfer OM`, `Transfer Site`, `Receive OM`, `Receive Site`, `Transfer Qty`, `Source Type`, `Destination Type`。

避免傳送：

- 全量 recommendations。
- Product Desc / Article Description（除非業務確認可傳）。
- 原始 DataFrame。

### 5.3 Auditor output schema

要求 JSON：

```json
{
  "risk_level": "low",
  "summary": "...",
  "warnings": [
    {"severity": "medium", "title": "來源集中", "detail": "...", "suggested_check": "..."}
  ],
  "positive_checks": ["..."]
}
```

Validation：

- `risk_level`: `low|medium|high`，其他降為 `low`。
- `warnings`: max 8。
- `severity`: `low|medium|high`。
- 每個文字欄位 truncate。
- 解析失敗：UI 不顯示錯誤堆疊，只顯示「AI 審計暫時不可用」。

### 5.4 UI 整合

在 `ui/display.py` 新增：

```python
def render_ai_audit_report(audit: dict): ...
```

建議 UI 位置：`app.py` line 200 quality check block 之後、`分析結果` 前。

流程：

1. 在 recommendations/statistics 存入 session 後，不立即強制呼叫 AI。
2. 在 quality status 顯示後提供 button：`🤖 執行 AI 邏輯審計`。
3. cache key 必須包含：
   - `current_run_key`
   - recommendation count / total qty
   - mode name
   - quality error count
4. 寫入：
   - `st.session_state['ai_audit_result']`
   - `st.session_state['ai_audit_key']`
5. 在 run key 變更時清除 AI audit session keys。

文案必須包含：

`AI 審計僅供參考，不會修改調貨建議；正式出貨前仍需按內部流程覆核。`

### 5.5 Auditor tests

新增 `tests/test_ai_auditor.py`：

1. payload 不含 full recommendations 長列表。
2. payload capped samples 不超過設定上限。
3. cross OM count 計算正確。
4. parser 支援純 JSON / fenced JSON。
5. invalid JSON fallback。
6. `audit_recommendations` 不 mutate 原 recommendations。
7. mock AI disabled/empty response 不 crash。

## 6. Phase 4 — Excel AI 分析摘要 Sheet

### 6.1 優先方案

不要先覆寫 `Notes`。新增一張 optional sheet：`AI分析摘要`。

原因：

- `services/notes.py` 已有精準 deterministic Notes。
- `tests/test_ai_note_simulation.py` 顯示逐行 AI Notes 可能只是 verbose paraphrase。
- Summary sheet 對管理層更有價值，也更省 token。

### 6.2 修改 `excel_generator.py`

修改 signature：

```python
def generate_excel_file(
    self,
    recommendations: List[Dict],
    statistics: Dict,
    output_path: Optional[str] = None,
    mode: str = None,
    ai_report: Optional[Dict] = None,
) -> bytes:
```

新增 method：

```python
def create_ai_analysis_sheet(self, writer, ai_report: Dict):
```

Sheet 內容建議：

- 標題：`AI分析摘要`
- 生成時間。
- AI 狀態 / 模型（不要顯示 API key）。
- Advisor 建議模式（如有）。
- Auditor risk level / summary。
- warnings table：`Severity`, `Title`, `Detail`, `Suggested Check`。
- positive checks。
- disclaimer：`AI內容僅供參考，不取代系統規則及人工覆核。`

若 `ai_report` 為 `None` 或空 dict，不新增 sheet，保持現有測試期待雙 sheet 仍可通過。

### 6.3 修改 `app.py`

Excel generation 時：

```python
ai_report = {
    'advisor': st.session_state.get('ai_advisor_result'),
    'audit': st.session_state.get('ai_audit_result'),
    'model': get_ai_status().get('model'),

    recommendations,

Excel cache key 必須加入 AI report hash，否則使用者先生成 Excel、再跑 AI audit 後下載可能仍是舊 Excel。

建議：

```

2. 有 `ai_report` 時包含 `AI分析摘要`。

### 11.2 手動 Streamlit 驗證

```powershell

```

情境：

   - app 可正常上傳、產生建議、下載 Excel。

## 12. 風險與緩解

| 風險 | 緩解 |
|---|---|
| Sidebar 在 upload 前 render，無法立即顯示資料感知 advisor | MVP 放主頁；sidebar 只讀 session_state 顯示上一輪結果 |
| LLM 回傳 invalid JSON | robust parser + fallback + 不阻塞 UI |
| LLM 推薦不存在模式 | 用 `mode_registry` validate，invalid 則不顯示推薦 |
| AI 審計幻覺 | UI 明確標示 advisory only；不修改 recommendations |
| 逐行 AI Notes 噪音/成本高 | 延後到 Phase 5，預設關閉；優先 summary sheet |
| API latency 影響 rerun | 使用 button 觸發 + session cache + timeout |
| 測試誤呼叫真 API | tests 全部 mock；`AI_ENABLED` 預設 false |
| Excel cache stale | `excel_run_key` 加入 AI report hash |

## 13. 建議給 DeepSeek V4 Pro 的執行提示

執行時請按以下順序落地，不要一次改完全部：

1. 先完成 `config.py`、`requirements.txt`、`services/ai_client.py`、`tests/test_ai_client.py`，跑相關測試。
2. 再完成 `services/ai_advisor.py`、`ui/display.py` advisor UI、`app.py` advisor button、`tests/test_ai_advisor.py`。
3. 再完成 `services/ai_auditor.py`、`ui/display.py` audit UI、`app.py` audit button、`tests/test_ai_auditor.py`。
4. 再完成 `excel_generator.py` AI sheet、`app.py` Excel cache key、`tests/test_excel_generator.py` 更新。
5. 最後更新 `README.md`、`VERSION.md`、`config.py VERSION`、`app.py` docstring。
6. 完整跑 `pytest tests/`。

不得做的事：

- 不要讓 AI output 進入 `TransferLogic.generate_transfer_recommendations(...)`。
- 不要上傳整份 DataFrame 或全量 recommendations。
- 不要在 log / UI 顯示 API key。
- 不要預設啟用 row-level note enhancer。
- 不要重構 sidebar/app 主流程以追求 advisor 位置；先用主頁 card，保留 sidebar session 顯示即可。
