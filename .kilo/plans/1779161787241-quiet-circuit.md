# 教學分頁設計計劃

## 目標
在現有 Streamlit 庫存調貨建議系統中新增一個「教學」分頁（Tab），以繁體中文為主，為每一種調貨模式提供圖例化教學，幫助使用者理解什麼情況下選擇什麼模式、調撥邏輯如何運作、調撥後的預期結果。

## 設計方案

### 架構：Streamlit Tabs 雙頁面

將 `app.py` 的主要內容區拆分為兩個 `st.tabs`：
1. **調貨系統** — 現有的上傳/分析/結果功能（原封不動遷入）
2. **模式教學** — 新增的教學分頁

### 教學分頁結構（`ui/tutorial.py`）

教學頁面將使用 **手風琴（expander）分組 + Mermaid 圖例 + 表格對比** 的方式呈現：

#### 1. 頂部導航區
- 簡介文字：「本教學頁面詳細說明 24 種調貨模式的適用場景、運作邏輯及預期結果」
- 一個模式快速跳轉 selectbox（選擇模式後滾動到對應 expander）

#### 2. 通用規則區塊（置頂）
以圖表展示所有模式共用的全局規則：
- ND 只轉出不接收
- 最高動銷店保護
- 轉出店不可同時接收
- 單件調貨後處理優化（Rebalance / Merge）
- 用 Mermaid flowchart 呈現全局流程

#### 3. 模式分組教學（8 個手風琴區塊）

按業務邏輯將 24 種模式分為 8 組，每組一個 expander：

| 分組 | 包含模式 | 主題 |
|------|---------|------|
| 基礎調貨 | A, B | 保守 vs 加強 |
| B 特別模式 | B2, B2a, B2L, B2La | Type=L 全轉出 / 保留2件 / T不出貨 |
| B 跨OM特別模式 | B3, B3a, B3L, B3La | 跨OM + Type + HD/Windy |
| 重點補0 | C, C1, C2 | 零庫存補貨系列 |
| 清貨模式 | D, D2 | ND 清貨 + 避免1件餘貨 |
| 強制轉出 | E1, E1b, E2 | ALL 標記強制調出 |
| 目標優化 | F, F2 | Target 導向分配 |
| ND/SKU 專項 | ND1, ND2, 精簡SKU(限同OM), 精簡SKU(跨OM) | ND互轉 + SKU精簡 |

#### 4. 每個模式的教學內容模板

每個模式在對應分組內使用一致的結構：

```
### 模式 [代碼]: [名稱]

**適用場景**: 什麼情況下使用（2-3句話）
**風險等級**: 低/中/高（視覺化標籤）

#### 流程圖 (Mermaid)
- 轉出篩選 flowchart：從所有店舖 → 條件判斷 → 成為轉出/跳過
- 接收篩選 flowchart：從所有店舖 → 條件判斷 → 接收類型分類
- 配對優先級：source → destination 的優先級匹配順序

#### 圖例說明
- 情境 A（典型場景）：輸入狀態 → 選擇此模式的原因 → 調撥過程 → 結果
- 情境 B（邊界場景）：如適用
- 用表格或 Mermaid 呈現「調撥前 → 調撥後」的數字變化

#### 與相關模式的差異
- 簡短對比表格（2-3 列）
```

#### 5. 模式選擇決策指南
在教學頁面底部加一個決策樹（Mermaid），幫助使用者根據自身情況選擇模式：
- 問題 1：是否需要清貨？→ D/D2
- 問題 2：是否有 Target 目標？→ F/F2
- 問題 3：是否需要強制轉出？→ E1/E1b/E2
- 問題 4：是否重點補零庫存？→ C/C1/C2
- 問題 5：是否有 Type 分類需求？→ B2 系列
- 問題 6：基礎調貨 → A/B

### 需要新增/修改的文件

| 文件 | 操作 | 說明 |
|------|------|------|
| `ui/tutorial.py` | **新增** | 教學分頁主要渲染模組 |
| `app.py` | **修改** | 將主內容區改為 `st.tabs(["調貨系統", "模式教學"])` |
| `ui/styles.py` | **修改** | 新增教學頁面專用的 CSS 樣式 |
| `static/styles.css` | **修改** | 新增教學相關 CSS class |

### 不修改的文件
- `business_logic.py` — 業務邏輯不變
- `ui/sidebar.py` — 側邊欄不變
- `ui/display.py` — 結果展示不變
- `strategies/*` — 策略代碼不變

### Mermaid 圖例規劃 → 改為 HTML/CSS 流程圖

每個模式至少包含：
1. **轉出篩選流程圖** — 用 HTML div 卡片 + CSS 箭頭展示店舖如何被判斷為轉出候選
2. **接收篩選流程圖** — 用 HTML div 卡片 + CSS 箭頭展示店舖如何被判斷為接收候選
3. **配對示意圖** — source type → destination type 的優先級（用帶顏色編號的箭頭連接）
4. **情境範例圖** — 具體數字案例，用 HTML 表格展示「調撥前 → 調撥後」

HTML 流程圖設計規範：
- 篩選節點用 `<div class="flow-node">` 配不同底色區分類型（ND=藍色, RF=綠色, 跳過=灰色）
- 箭頭用 CSS `::after` 偽元素或 `↓` 字符
- 條件判斷用菱形外框 `<div class="flow-decision">`
- 結果用 `<div class="flow-result">` 配綠色/紅色邊框

### 實施步驟

1. **Step 1**: 新增 `ui/tutorial.py`，包含所有教學內容渲染函數
   - `_render_global_rules()` — 通用規則（HTML 流程圖）
   - `_build_flow_chart(nodes)` — HTML 流程圖生成輔助函數
   - `_build_scenario_table(before, after)` — 情境對比表格生成
   - `_render_mode_tutorial(mode_code)` — 單一模式教學（含流程圖+情境）
   - `_render_group_tutorial(group_name, modes)` — 分組教學容器
   - `_render_decision_guide()` — 模式選擇決策樹（HTML 流程圖）
   - `render_tutorial_page()` — 教學頁面主函數

2. **Step 2**: 修改 `app.py`，用 `st.tabs()` 包裹現有內容 + 教學頁面

3. **Step 3**: 在 `static/styles.css` 中新增教學頁面樣式（卡片、標籤、流程圖容器等）

4. **Step 4**: 測試所有 24 個模式的教學內容渲染

### 技術決策（已確認）

- **圖表渲染**: 使用 HTML + inline CSS 手繪流程圖卡片和箭頭，無需額外套件
- **頁面結構**: 使用 `st.tabs(["調貨系統", "模式教學"])` 頂部 Tab 分頁
- **性能**: 教學頁面內容靜態，不涉及數據處理
- **回應式**: 教學頁面在寬屏下使用 columns 排列流程圖和說明
