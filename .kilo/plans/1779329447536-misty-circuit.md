### 4d. 重構 `business_logic.py.__init__` 的模式常數與衍生集合

刪除 `self.mode_a = "保守轉貨"` ... `self.mode_simplified_sku_cross = "精簡SKU(跨OM)"` 等 24 行常數，
改為從 registry 動態生成：

```python
from models.mode_registry import REGISTRY, get_mode_families

class TransferLogic:
    def __init__(self, ...):
        # 從 registry 動態生成模式常數
        for _name, _def in REGISTRY.items():
            attr_name = f"mode_{_def.code.lower().replace('(', '').replace(')', '').replace(' ', '_')}"
            # 簡化：直接用 name 作為實例屬性
            setattr(self, f"mode_{_def.code}", _def.name)  # 不對，code 有中文
        
        # 更好的方案：保留 self.mode_a = "保守轉貨" 等，但用 registry 驅動
        self._mode_defs = REGISTRY
        self._mode_by_name = {d.name: d for d in REGISTRY.values()}
```

> **設計決策**：`self.mode_a` 等常數被策略類別直接比較（`mode == self.mode_a`），不能輕易改名。保留這些常數但改為從 registry 自動設定。

實際做法：在 `__init__` 頂部，遍歷 `REGISTRY` 並用 `code` 的正規化版本作為屬性名：

```python
self._mode_defs = {d.name: d for d in MODE_DEFS}
self._mode_by_code = {d.code: d for d in MODE_DEFS}
# 動態設定 mode 常數（向後兼容）
self.mode_a = self._mode_by_code["A"].name
self.mode_b = self._mode_by_code["B"].name
# ...但 code→attr 的映射需要硬編碼或正規化
```

> **風險評估**：code→attribute 的映射（`"A"` → `self.mode_a`，`"B2a"` → `self.mode_b_special_a`，`"精簡SKU(跨OM)"` → `self.mode_simplified_sku_cross`）目前是手動的。若要自動化，需要在 `ModeDef` 加一個 `attr_name` 欄位，或改為統一用 `self._mode_by_code["A"]` 查詢。
>
> **推薦方案**：在 `ModeDef` 加 `attr_name: str` 欄位（如 `"mode_a"`, `"mode_b_special"`, `"mode_simplified_sku_cross"`），`__init__` 用 `setattr(self, d.attr_name, d.name)` 自動設定。這樣策略中的 `mode == self.mode_a` 比較不變。

#### 5b. 新增可選計時 decorator（`services/perf_timer.py`）

```python
import os, time, logging
from functools import wraps

ENABLE_PERF = os.environ.get('KILO_PERF_TIMING', '').lower() in ('1', 'true')

def perf_timer(label: str):
    def decorator(fn):
        if not ENABLE_PERF:
            return fn
        @wraps(fn)
        def wrapper(*args, **kwargs):
            t0 = time.perf_counter()
            result = fn(*args, **kwargs)
            elapsed = time.perf_counter() - t0
            logging.getLogger('perf').info(f"[PERF] {label}: {elapsed:.3f}s")
            return result
        return wrapper
    return decorator
```

在 `generate_transfer_recommendations` 和 `match_by_priority` 加上 `@perf_timer` 裝飾，
預設關閉（`KILO_PERF_TIMING=1` 才啟用），零效能損耗。

1. 在 `models/mode_registry.py` 的 `MODE_DEFS` 加 1 筆 `ModeDef(...)`
2. 如果需要新策略，建立 `strategies/new_mode.py` 並在 `_init_strategies()` 註冊
3. 如果需要新的 source/dest 邏輯，在 `business_logic.py` 加對應方法並設 `source_method`/`dest_method`
4. 更新文檔（README/tutorial/VERSION/調貨模式詳解/AI brief）
5. 完成——不需再修改 `_ALL_MODES`、`_CROSS_OM_*`、`_MODE_OPTIONS`、`MODE_NAME_MAP` 等 15+ 個白名單
