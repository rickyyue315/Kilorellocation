# F2: 避免Target店舖成為轉出店舖

## Bug Description

在F2模式中，現有邏輯僅在**逐Article層級**檢查Target值來跳過轉出源識別：若某店舖在該Article有`Target > 0`，則不加入轉出源。

但當以下情境發生時會出現Bug：
- **店舖X** 在 **Article A** 有 `Target = 5` → 應接收
- 但沒有其他店舖可轉貨給 Article A 的店舖X
- **店舖X** 在 **Article B** 沒有 Target（空白/0）→ 現有逐Article檢查**不會跳過**它
- 結果：**店舖X** 在 Article B 變成轉出源，轉貨給其他Target店舖Y

**根本問題**：逐Article檢查不足。店舖X是「Target店舖」（在任何Article有Target>0），不應在任何Article成為轉出源。

## 修正方案

在F2模式下，預先計算**全資料集中所有有Target>0的店舖集合**，然後在`identify_sources`的F/F2分支中，若店舖屬於此集合，則跳過不加入轉出源。

### 修改檔案

**`business_logic.py`**

#### 1. `generate_transfer_recommendations` 方法

在主要Article分組loop之前，預先計算全域Target店舖集合：

```python
target_stores = set()
if mode == self.mode_f_target_only:
    target_series_full = self._parse_target_series(df)
    if 'Target' in df.columns:
        target_mask = target_series_full > 0
        target_stores = set(
            df.loc[target_mask.fillna(False), 'Site']
            .astype(str).str.strip().str.upper()
        )
```

在呼叫`identify_sources`時傳入：

```python
sources = self.identify_sources(
    group_df, mode, 
    protected_sites=target_stores if mode == self.mode_f_target_only else None
)
```

#### 2. `identify_sources` 方法

- 簽名新增 `protected_sites: Optional[Set[str]] = None` 參數
- 在 F/F2 模式的 ND loop 和 RF loop 中，在既有 Target 檢查之後，新增一個全域保護檢查：

```python
# F2模式：若店舖在任何Article有Target>0，則在所有Article都不作為轉出源
if mode == self.mode_f_target_only and protected_sites:
    site_key = str(row['Site']).strip().upper()
    if site_key in protected_sites:
        continue
```

### 不變的邏輯

- F模式不受影響（`protected_sites` 只對 `mode_f_target_only` 傳入）
- 識別接收端 (`identify_destinations`) 不需修改—Target店舖已在F2模式下正確處理
- 匹配邏輯 (`_match_transfers_f_mode`) 不需修改—只要來源清單已正確過濾即可
- 所有測試不應受影響（現有測試未測試跨Article的保護情境）

### 驗證

1. 執行現有測試套件確保無迴歸
2. 手動驗證情境：店舖X在Article A有Target=5，在Article B無Target → 不應在Article B的推薦中出現在轉出源欄位
